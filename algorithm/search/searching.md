# searching

## linear search
요소들을 순서대로 검사합니다. 이 요소들은 직선으로 늘어서 있습니다.

## binary search
요소의 중앙(target)부터 key값을 비교하기 시작합니다. 비교할 때마다 요소들은 적어집니다.
- key보다 작을경우 -> target은 최솟값이 됨
- key보다 클 경우 -> target은 최댓값이 됨

위 비교연산을 한 후 탐색 범위를 다시 설정하고, key가 같을 때까지 비교를 반복합니다.
- 데이터가 이미 정렬(오름차순/내림차순)되어 있어야 합니다. 
- 연결리스트에서는 효율이 나오지 않습니다. 
 linear search보다 빠릅니다.
 
 `시간 복잡도: O(logN)` : 
 단계를 진행할 때마다 탐색 범위가 절반씩 줄어듭니다. 따라서 비교연산의 기댓값은 logN입니다. 
 최악의 경우에도 비교횟수는 logN+1회가 나옵니다. 
 logarithm에 관하여-> 단계를 진행할 때마다 N에 N/2를 대입합니다. 그리고 총 단계의 개수는 logN개입니다. 마지막 비교단계에서 key가 맞는지 확인하는 연산 1회를 추가로 진행합니다.
